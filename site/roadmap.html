Despite Qpackt being barely an MVP right now, I have some great plans for it. I imagine it can be a one-stop-shop for web deployment and monitoring. Before I get there though, there are many steps in between. Here's a basic roadmap for the next year or so:

  1. Get 100 stars on GitHub and gather feedback about missing features. I'd like to see some interest and start coding features that are actually needed. There is nothing worse than spending months creating something nobody wants. This phase can take weeks or months, obviously.

  2. Make it production-ready. I mean, it works as it is (if you can read it), but it's quite far from what I consider "good." Things that need more love: code quality (eliminate unwraps, achieve at least 70% mutation testing coverage), performance acceptance tests, better deployment options (maybe create a Docker image), and user support (via Discord or something else). Probably doable in 2-3 months.

  3. Once it's production-ready, Qpackt will enter a stabilization period. Hopefully, more people will use it, and they will provide feedback on what could be done differently. This should take around 6 months. After this, every version will be LTS. A new minor version should be released every few weeks. A new major version should be released about every year initially, with decreasing frequency over time.

Follow me on Twitter if you want to know about the progress.
